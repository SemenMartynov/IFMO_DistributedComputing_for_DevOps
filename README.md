## Мониторинг с Prometheus и Grafana

Для настройки мониторинга системы используется отдельный плейбук, который не влияет на работу продакшн-окружения WordPress.

### Запуск мониторинга

```bash
cd ansible
ansible-playbook -i inventory.ini monitoring-playbook.yml
```

### Доступ к мониторингу

После успешного выполнения плейбука, мониторинг будет доступен по следующим адресам:

- **Prometheus**: http://51.250.45.157:9090
- **Grafana**: http://51.250.45.157:3000 (логин: admin, пароль: admin)
- **cAdvisor**: http://51.250.45.157:8081

### Возможности мониторинга

1. **Системные метрики**:
   - Загрузка CPU
   - Использование памяти
   - Дисковое пространство
   - Сетевой трафик

2. **Метрики контейнеров**:
   - Использование ресурсов контейнерами
   - Статус контейнеров

3. **Метрики MySQL**:
   - Количество запросов
   - Производительность запросов
   - Репликация

### Преимущества отдельного плейбука для мониторинга

1. **Изоляция от продакшн-окружения** - мониторинг запускается отдельно и не влияет на работу WordPress
2. **Независимое обновление** - можно обновлять мониторинг без перезапуска основных сервисов
3. **Гибкость настройки** - можно настраивать мониторинг под конкретные потребности
